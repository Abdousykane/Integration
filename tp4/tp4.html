TP4 – Exercices 1 et 3

Exercice 1

Commentaire – Exercice 1

À partir d'un fichier HTML intouchable contenant 3 blocs, il fallait réaliser une mise en page avec un menu latéral fixe (position: fixed) et créer une interaction où le clic sur le menu change les couleurs et affiche le paragraphe correspondant.

Réalisation et Difficultés : La première étape consistait à libérer de l'espace à gauche (margin-left) pour y placer le menu en position fixe. La vraie difficulté de cet exercice résidait dans l'interdiction de modifier le HTML : Pour les couleurs : J'ai dû utiliser des sélecteurs structurels comme :nth-child ou :nth-of-type pour cibler le 1er lien, le 1er titre et le 1er paragraphe afin de leur attribuer la même couleur, et ainsi de suite pour les suivants. Pour l'interaction : Le plus dur a été de comprendre comment masquer les paragraphes et ne faire apparaître que celui désiré. J'ai utilisé la pseudo-classe :target. Cela permet de modifier le style de l'élément ciblé par l'ancre de l'URL (le #id).

C'était un exercice technique très intéressant qui m'a montré qu'on peut créer une navigation fonctionnelle (système d'onglets) uniquement en CSS, sans avoir besoin de JavaScript, juste en maîtrisant bien les sélecteurs.

Exercice 3

Commentaire – Exercice 3

L'objectif était de styliser un tableau HTML brut pour qu'il corresponde à une capture d'écran fournie, avec une contrainte forte : interdiction absolue de modifier le code HTML. Il fallait également minimiser le nombre de règles CSS.

Puisque je ne pouvais pas ajouter de classes dans le HTML (comme .ligne-paire ou .colonne-importante), j'ai dû utiliser exclusivement les pseudo-classes structurelles CSS : :nth-child(odd) et :nth-child(even) pour créer l'effet "zèbré" (une ligne sur deux colorée), ce qui rend le tableau plus lisible. J'ai dû utiliser des formules plus précises comme :nth-child(n+2) ou cibler des colonnes spécifiques pour reproduire la mise en forme des entêtes et des données.

Cet exercice ressemblait presque à des mathématiques ! Au début, c'était un peu abstrait de devoir "compter" les éléments via le CSS, mais j'ai compris la puissance de cette méthode. Elle permet de garder un code HTML très propre et de gérer la mise en forme de données dynamiques (comme un tableau dont on ne connaît pas la longueur à l'avance) sans avoir à classer manuellement chaque ligne.
